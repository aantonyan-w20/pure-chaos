# Pure Chaos — Tic Tac Toe Park

A Roblox game built by two friends. Floating islands in the sky, connected by
wooden bridges, each with a tic-tac-toe table. Sit in the **red chair** to play
X, the **blue chair** to play O — when both chairs fill up, the cameras cut to
a cinematic "you VS them" intro and lock to a top-down view of the board. A
hidden portal leads to a mystery island where the rules flip: making three in
a row **loses** (misère tic-tac-toe).

The whole park is generated by code at runtime — there is no hand-built map.
All game code lives in this repository as `.luau` files, managed with
[Rojo](https://rojo.space/), and shared through git/GitHub.

## Project structure

```
src/
  shared/
    Config.luau              every tunable number/color in the game
  server/
    Main.server.luau         entry point: builds the park at startup
    World/                   island, bridge, portal, scenery, sky builders
    TicTacToe/
      Logic.luau             pure game rules (no Roblox instances)
      Board.luau             the physical clickable 3x3 board + status sign
      Furniture.luau         table + the assigned X/O chairs
      MatchWatcher.luau      fires the match start/stop event to players
      GameTable.luau         glues all of the above into one playable table
  client/
    MatchCamera.client.luau  cinematic camera when a match starts
    VsBanner.luau            the "name VS name" banner
default.project.json         Rojo mapping: which folder goes where in the game
CLAUDE.md                    coding rules — read before changing anything
```

Rojo places everything automatically: you never copy-paste scripts into
Studio. `src/server` becomes `ServerScriptService.Server`, `src/client` becomes
`StarterPlayerScripts.Client`, `src/shared` becomes `ReplicatedStorage.Shared`.

## One-time setup (each person)

1. Install [Roblox Studio](https://create.roblox.com/) and log in.
2. Install Rojo:
   - **macOS:** `brew install rojo`
   - **Windows/any:** install [Rokit](https://github.com/rojo-rbx/rokit), then run `rokit install` in this folder.
3. Install the Rojo plugin for Studio: run `rojo plugin install` in a terminal,
   then restart Studio (a Rojo button appears in the Plugins tab).
4. Clone this repository:
   ```sh
   git clone git@github.com:aantonyan-w20/pure-chaos.git
   cd pure-chaos
   ```

## Daily workflow

1. Pull the latest changes first:
   ```sh
   git pull
   ```
2. Start the Rojo sync server in this folder:
   ```sh
   rojo serve
   ```
3. Build a fresh place and open it in Studio:
   ```sh
   rojo build -o game.rbxl && open game.rbxl
   ```
4. In Studio, click the **Rojo** plugin button → **Connect**. Now edit any
   `.luau` file in your editor, save, and it updates in Studio instantly.
5. Press **Play** to test. To test a full match (two players in chairs), use
   Studio's **Test → Clients and Servers → 2 Players → Start**.
6. Share your work:
   ```sh
   git add -A
   git commit -m "describe what you changed"
   git push
   ```

## Ground rules so we don't clobber each other

- **Always `git pull` before you start working.**
- Commit and push small changes often — big rare pushes cause painful merge conflicts.
- Follow the coding rules in [CLAUDE.md](CLAUDE.md) — file/function size limits,
  where settings live, how modules are split. AI agents read that file too.
- Run `rojo build -o game.rbxl` before pushing to make sure the project compiles.
- If you both need to change the same file, talk first (or use branches).
